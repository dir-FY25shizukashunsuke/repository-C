# Agents.md - Project Overview & AI Context

このファイルは、AI エージェントがリポジトリの全体像を把握し、より正確なドキュメント（README.md等）を生成するためのコンテキストを提供します。

## プロジェクトの目的
`repository-C` は、ユーザー管理機能を持つシステムのコアリポジトリです。サブモジュールとしてのプロトタイプ実装（`repository-A`）と、プロダクション品質のユーザー管理モジュール（`user_management`）を統合しています。

## リポジトリ構造
- **`repository-A/` (Submodule)**: 
    - `app.py`: Flask を使用した API の初期プロトタイプ。
    - API エンドポイントや基本的なルーティングの定義が含まれます。
- **`user_management/`**: 
    - クリーンな設計を目指したユーザー管理のロジック層。
    - `user_manager.py`: ユーザーの追加、削除、検索など、ビジネスロジックの中核。
- **`scripts/`**: 
    - `agent_readme.py`: GitHub Copilot SDK を活用した AI ドキュメント生成エージェント。

## AI オートメーション戦略
このプロジェクトでは、ドキュメントの鮮度を保つために AI による自動更新を採用しています。
- **`SKILL.md`**: README の出力形式や AI の思考ルールを定義。
- **`Agents.md` (このファイル)**: 設計思想、リポジトリの歴史、AI に汲み取ってほしい「行間」の文脈を記録。

## 開発者へのメッセージ
ソースコードのコメントに書ききれない設計の意図や、将来的な拡張予定などは、この `Agents.md` に追記してください。AI がその内容を自動的に README に反映させ、常に最新の「プロジェクトの地図」を維持します。

## 設計思想・拡張計画（AI用コンテキスト）

- クリーンアーキテクチャを採用し、ビジネスロジックとインフラ層を明確に分離。
- 型安全性（Python: dataclass/type hints, TypeScript: types/interfaces）を重視。
- ユーザー管理はイミュータブル返却を基本とし、副作用を最小化。
- セキュリティ（パスワードハッシュ化、SQLインジェクション対策）を徹底。
- サブモジュールの拡張や追加が容易な構造。
- ドキュメント自動生成・自動更新の仕組みを標準化。
- 今後の拡張例：
    - OAuth認証や外部ID連携
    - 管理者権限・ロール管理
    - 多言語対応
    - APIレスポンスの型保証
    - CI/CDによるテスト自動化
