# Agents.md - Project Overview & AI Context

このファイルは、AI エージェントがリポジトリの全体像を把握し、より正確なドキュメント（機能定義書.md、シーケンス図.md等）を生成するためのコンテキストを提供します。

## プロジェクトの目的
`repository-C` は、複数サブモジュール（例：`repository-A`、`repository-B`など）の情報を自動取得し、統合的な資料（**機能定義書.md**、**シーケンス図.md**）を生成・管理するための専用リポジトリです。

## リポジトリ構造


- **`repository-A/` (Submodule)**: 
    - `app.py`: Flask を使用した API の初期プロトタイプ。
    - API エンドポイントや基本的なルーティングの定義が含まれます。

- **`repository-B/` (Submodule)**: 
    - サブモジュールBの主要な構成・API等（今後拡張予定）


## AI オートメーション戦略

このプロジェクトでは、複数サブモジュールの最新情報をもとにAIが以下の資料を自動生成・統合・更新します：
- **`機能定義書.md`**: APIエンドポイント、データモデル、セキュリティ機能などの統合ドキュメント
- **`シーケンス図.md`**: 処理フロー、API呼び出しシーケンスのMermaid図

## 開発者へのメッセージ

ソースコードのコメントに書ききれない設計の意図や、資料生成・自動化の運用方針などは、この `Agents.md` に追記してください。AI がその内容を自動的に機能定義書・シーケンス図に反映させ、常に最新の「プロジェクトの地図」を維持します。

---
## 設計思想・拡張計画（AI用コンテキスト）

- 複数サブモジュールの情報を正確に取得し、以下の資料を自動生成・自動更新・統合する仕組みを標準化：
  - **機能定義書.md**: APIエンドポイント、データモデル、セキュリティ機能
  - **シーケンス図.md**: 処理フロー、Mermaid記法のシーケンス図
- サブモジュールの拡張や追加が容易な構造。
- ドキュメント自動生成・自動更新の仕組みを標準化。
- 今後の拡張例：
    - サブモジュールの多段管理
    - 複数サブモジュール・複数リポジトリ横断の資料生成
    - サブモジュールごとの個別資料と統合資料の自動生成
    - アーキテクチャ図、ER図の自動生成
    - 多言語対応
    - CI/CDによる資料自動検証

---
## 具体的なAI反映例

- 機能定義書には「設計思想」「拡張計画」「サブモジュール管理方針」を必ず明記すること。
- サブモジュールや主要モジュールの役割を簡潔にまとめること。
- 複数サブモジュールのAPIや構成の表形式まとめを推奨。
- シーケンス図には正常系・異常系の両方のフローを図示すること。
- Mermaid記法を使用し、可読性の高い図を生成すること。
