# SKILL.md - README生成ルール

## 推奨される README フォーマット

````markdown
# プロジェクト概要

- 目的・特徴
- ディレクトリ構成（ツリー形式）
- サブモジュール・主要モジュールの説明
- APIエンドポイント一覧（表形式）
- UserManager等の主要クラス・メソッド一覧（表形式）
- クイックスタート（インストール・起動・API例）
- 技術スタック
- 設計思想・拡張計画
- 最終更新日時
````

## AIへの指示
- Agents.mdの内容を必ず反映すること
- コードやdocstringからAPIやクラス情報を抽出し、表形式でまとめること
- セキュリティ・設計思想・拡張予定は強調して記載すること
- サブモジュールのREADMEも参考にすること
